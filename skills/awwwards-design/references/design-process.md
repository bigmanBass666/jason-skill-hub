## 1. 设计流程总览

```
理解与定义 → 视觉方向 → 交互架构 → 实现与打磨 → 质检与交付
    ↑                                              │
    └──────────── 迭代反馈 ────────────────────────┘
```

**关键原则：** 每个阶段的输出是下一个阶段的输入。不要跳过"理解与定义"直接写代码——这是大多数项目失败的原因。

---

## 阶段一：理解与定义

### 1.1 明确项目目标

在动手之前，回答这些问题：

| 问题 | 为什么重要 |
|------|-----------|
| 这个网站的核心目的是什么？ | 目的决定了设计方向——品牌展示 ≠ 产品销售 ≠ 内容阅读 |
| 目标用户是谁？ | 用户画像影响字体大小、交互复杂度、色彩饱和度 |
| 用户应该在离开后记住什么？ | 这就是 Wow Moment 的方向 |
| 有哪些内容/资产可以使用？ | 真实内容 > AI 生成 > 占位符。没有真实内容就先设计框架 |
| 技术约束是什么？ | 3D 需要更多开发时间，WebGL 有设备兼容性问题 |

### 1.2 定义叙事核心

用一个句子定义这个网站的"故事"：

- ❌ "为公司做一个好看的网站"
- ✅ "让访客感受到速度与精准——像驾驶一台精密仪器"
- ✅ "在数字空间中漫步，发现创意的每一个角落"
- ✅ "把品牌百年的历史压缩进一次滚动体验"

叙事核心决定：
- 色彩倾向（温暖/冷峻/中性）
- 动画节奏（快速干脆/缓慢流畅）
- 交互密度（极简/丰富）
- 信息密度（稀疏/密集）

### 1.3 信息架构

根据叙事核心组织内容。内容组织服务叙事，而非功能清单：

```
首页（钩子）→ 核心展示（证明）→ 深度内容（信任）→ 联系/行动（转化）
```

## 阶段二：视觉方向

### 2.1 选择视觉风格

基于叙事核心和项目类型，从 SKILL.md 中列出的 8 种风格中选择一种。

**选择原则：**
- 风格服务于叙事，不是为了炫酷
- 一种风格的深度执行 > 多种风格的浅尝辄止
- 考虑品牌属性——奢侈品牌 ≠ 科技公司 ≠ 创意机构

### 2.2 建立设计系统

在视觉方向确定后，立即建立完整的设计系统：

1. **色彩变量** — CSS Custom Properties，不超过 5 个核心色
2. **字体系统** — 展示字体 + 正文字体 + 等宽字体（如需要）
3. **间距系统** — 使用 `clamp()` 实现流式缩放，定义 4-5 个间距级别
4. **组件变量** — 圆角、阴影、边框、过渡时长
5. **动画变量** — 缓动曲线、时长、延迟

```css
:root {
  /* 色彩 */
  --c-bg: #0a0a0a;
  --c-text: #f0ece4;
  --c-accent: #ff3d00;
  --c-muted: #666;

  /* 字体 */
  --font-display: 'Clash Display', sans-serif;
  --font-body: 'Satoshi', sans-serif;

  /* 间距 */
  --space-xs: clamp(4px, 0.5vw, 8px);
  --space-sm: clamp(8px, 1vw, 16px);
  --space-md: clamp(16px, 2vw, 32px);
  --space-lg: clamp(32px, 4vw, 64px);
  --space-xl: clamp(64px, 8vw, 128px);

  /* 动画 */
  --ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-out-back: cubic-bezier(0.34, 1.56, 0.64, 1);
  --duration-fast: 0.3s;
  --duration-normal: 0.6s;
  --duration-slow: 1.2s;
}
```

## 阶段三：交互架构

### 3.1 定义交互层级

不是每个网站都需要所有交互模式。根据项目复杂度和目标选择：

| 交互层级 | 适用场景 | 模式 |
|---------|---------|------|
| **基础层**（所有项目） | 任何网站 | 自定义光标、页面加载动画、滚动揭示、悬停反馈 |
| **增强层**（推荐） | 品牌展示、作品集 | + 页面过渡、磁性按钮、文字揭示动画 |
| **高级层**（差异化） | 创意机构、大型品牌 | + 3D/WebGL、实验性导航、Wow Moment |
| **顶级层**（头奖级） | 追求极致的项目 | + 全部模式 + 签名式交互创新 |

### 3.2 设计 Wow Moment

参考 SKILL.md 中的 Wow Moment 设计步骤。确保：
- 它出现在用户进入网站的前 10 秒内
- 它与叙事核心一致（不是硬塞的特效）
- 它在移动端有合理的降级方案
- 它不会显著影响页面加载性能

### 3.3 移动端交互规划

移动端不是事后补丁——它是独立的设计：

| 桌面模式 | 移动端替代 |
|---------|-----------|
| 自定义光标 | 触摸涟漪反馈 |
| 悬停揭示 | 点击/长按触发 |
| 水平滚动画廊 | 垂直滑动或轮播 |
| 3D 场景 | 2D 预渲染视频或简化版本 |
| 鼠标跟随效果 | 设备倾斜/陀螺仪响应 |

## 阶段四：实现与打磨

### 4.1 实现顺序

推荐的开发顺序（从底层到表层）：

1. **基础框架** — CSS 变量、布局系统、字体加载
2. **静态页面** — 语义化 HTML、内容结构、响应式布局
3. **基础动画** — 滚动揭示、悬停效果、页面加载
4. **进阶交互** — 自定义光标、页面过渡、磁性按钮
5. **Wow Moment** — 核心亮点效果
6. **3D/WebGL**（如需要）— 最后实现，因为性能影响最大
7. **性能优化** — 懒加载、设备检测、Lighthouse 审计

### 4.2 打磨清单

动画打磨不是调参数——它是对时间和空间的精确控制：

- **入场时机**：元素出现在可视区域后多久开始动画？（通常 0-200ms）
- **错开节奏**：多个元素依次入场的时间间隔（通常 50-150ms）
- **缓动选择**：快速入场用 Expo Out，悬停用 Back Out（带回弹）
- **留白呼吸**：动画元素之间有足够的视觉空间
- **状态反馈**：每个可交互元素都有 hover/active/focus 状态
- **加载体验**：有优雅的加载过程，不是白屏等待

## 阶段五：质检与交付

### 5.1 第一印象测试（5 秒法则）

打开网站全屏截图，只看 5 秒：
- 这个网站在讲什么？（叙事清晰度）
- 我想继续看下去吗？（好奇心驱动）
- 它看起来像模板吗？（原创性）

### 5.2 10 秒交互测试

在网站上操作 10 秒：
- 导航是否直观？
- 动画是否流畅？
- 有没有让我想截图的瞬间？

### 5.3 质量自检

使用 `references/quality-framework.md` 中的四维自检框架逐项检查。

### 5.4 Lighthouse 审计

Chrome DevTools Lighthouse（Mobile 模式）目标：
- Performance ≥ 90
- Accessibility ≥ 80
- Best Practices ≥ 90
- SEO ≥ 90

## 项目类型适配指南

不同类型的网站有不同的设计侧重点。以下是基于数百个 Awwwards SOTD 头奖网站的统计分析得出的适配策略：

### 作品集 / 个人网站（Portfolio）

**侧重点：** 创意 > 视觉 > 可用性 > 内容

- Wow Moment 是重中之重——评委/访客需要在 5 秒内被震撼
- 交互可以更大胆，因为访客预期的是创意体验
- 作品展示是核心，确保作品图片/视频质量极高
- 「关于我」页面可以用叙事滚动讲述个人故事
- 推荐风格：新极简主义、数字朋克、反设计

**典型结构：**
```
Hero（Wow Moment）→ 精选作品（交互画廊）→ 关于（叙事滚动）→ 联系
```

### 品牌官网 / 产品页（Brand / Product）

**侧重点：** 视觉 = 可用性 > 创意 > 内容

- 品牌一致性是第一要务——每个像素都要符合品牌调性
- 产品展示需要让用户"感受到"产品（3D 旋转、视频、交互）
- 导航必须直观——用户需要快速找到产品/购买信息
- CTA 设计要突出但不突兀
- 推荐风格：电影沉浸、新极简主义、Bento Grid

**典型结构：**
```
Hero（品牌宣言）→ 产品展示（3D/视频）→ 特性（Bento Grid）→ 评价/数据 → CTA
```

### 创意机构 / 设计工作室（Agency / Studio）

**侧重点：** 创意 = 视觉 > 可用性 > 内容

- 网站本身就是最好的"作品"——展示能力的第一载体
- 导航可以实验性，但必须能被理解（3 秒内）
- 案例展示需要沉浸式体验（点击进入全屏案例页）
- 团队介绍可以用创意方式呈现（非传统头像列表）
- 推荐风格：任何风格，关键是执行的深度和原创性

**典型结构：**
```
Hero（创意宣言/Wow Moment）→ 精选案例（交互画廊）→ 服务/方法 → 团队 → 联系
```

### SaaS / 技术产品

**侧重点：** 可用性 > 视觉 > 内容 > 创意

- 功能清晰传达比视觉震撼更重要
- Bento Grid 是展示功能特性的最佳布局
- 数据可视化（图表、计数器）增加说服力
- 交互需要优雅但不能影响效率
- 推荐风格：Bento Grid、新极简主义、档案式排版

**典型结构：**
```
Hero（价值主张 + CTA）→ 社会证明（Logo/数据）→ 功能（Bento Grid）→ 定价 → CTA
```

### 品牌故事 / 长页（Brand Story / Long-form）

**侧重点：** 创意 = 内容 > 视觉 > 可用性

- Scrollytelling 是核心交互——滚动就是叙事的节奏
- 内容品质是关键——文案需要精心打磨
- 全屏图片/视频与文字交替出现
- 可以包含数据可视化、时间线等叙事工具
- 推荐风格：电影沉浸、有机感官、新极简主义

**典型结构：**
```
开篇（钩子）→ 第一章 → 第二章 → ... → 结尾（CTA）
```
