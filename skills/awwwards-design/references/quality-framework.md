# 设计质量框架

基于 Awwwards 2024–2026 年度 SOTD 头奖网站逆向工程，提炼出的设计质量自检体系。这四个维度构成了「世界级设计」的完整图景——用它们作为自查清单，衡量你的设计是否达到了 Awwwards 级别的水准。

---

## 质量优先级矩阵

当一个设计决策需要取舍时（比如"要加这个炫酷效果但可能影响加载速度吗？"），用这个优先级指导投入方向。权重来自对数百个 Awwwards SOTD 头奖网站的分析——视觉和可用性合计 70%，这是不可妥协的基础。

| 维度 | 权重 | 核心关注 | 常见陷阱 |
|------|------|----------|----------|
| **Design 设计** | 40% | 视觉层次、字体品位、色彩克制、间距节奏、原创性 | 使用模板/通用字体/素材库图片会让网站看起来"千篇一律" |
| **Usability 可用性** | 30% | 导航直观、加载 <3s、移动端体验、Core Web Vitals | 一个好看但 5 秒加载的网站，用户不会等 |
| **Creativity 创意** | 20% | **一个**签名式「Wow Moment」、交互创新、形式统一内容 | 20 个普通特效 ≠ 1 个令人难忘的时刻 |
| **Content 内容** | 10% | 真实文案、品牌声音、原创媒体 | Lorem ipsum 和素材图会让品质感归零 |

> **为什么是这个权重？** 视觉（40%）+ 可用性（30%）= 70%。这意味着一个网站首先必须好看且好用，这是入场券。创意（20%）是从「还不错」到「令人难忘」的差异化因素——没有它，网站只是精致但容易被遗忘。内容（10%）是最容易被忽视但能显著提升整体品质感的环节——真实、有品牌声音的文案能让设计从"漂亮"升级为"专业"。

## Design 设计（40%）— 自检清单

### 视觉层次
- [ ] 页面打开的瞬间，眼睛知道先看哪里（明确的视觉锚点）
- [ ] 信息层级不超过 3 级（主 → 次 → 辅助），每一级在字号/颜色/粗细上有明显区分
- [ ] 没有任何两个同等重要的元素在争夺注意力

### 字体
- [ ] 展示字体独特且有意义（不是 Inter/Roboto/Arial/Poppins ——这些字体质量很好但已经被过度使用到失去了视觉个性）
- [ ] Hero 字号极端化（`clamp(4rem, 12vw, 14rem)`），字体本身就是视觉元素
- [ ] 字体配对有对比且和谐（展示字体 vs 正文字体风格差异明显）
- [ ] 行高、字间距经过调整（Hero 行高 0.85-0.92，字间距 -0.03em 到 -0.05em）

### 色彩
- [ ] 调色板不超过 3 色（背景 + 文字 + 1个强调色）——克制本身就是高级感
- [ ] 强调色使用克制，仅出现在关键交互点和品牌元素上
- [ ] 色彩选择服务于品牌叙事，而非随意搭配
- [ ] 暗色模式下 3D 元素和渐变效果得到充分展示

### 间距与节奏
- [ ] 每个间距都是刻意设计的（不是默认 margin/padding）
- [ ] 章节间距使用 clamp 实现流式缩放
- [ ] 留白是有意义的（Ma/間 的哲学）——留白是内容而非空白
- [ ] 元素对齐使用一致的网格系统

### 原创性
- [ ] 至少有一个视觉元素不是来自模板或素材库
- [ ] 图形元素服务于叙事概念
- [ ] 整体视觉感觉「做过设计」而非「选过模板」
- [ ] 与同类型网站放在一起，能一眼区分出来

### 品质信号（CSS 零成本品质提升）
- [ ] **自定义 `::selection` 选中色**（使用 var(--c-accent)）
- [ ] **自定义光标**（含移动端降级）
- [ ] **字体抗锯齿**（-webkit-font-smoothing: antialiased）
- [ ] **自定义滚动条**
- [ ] **系统光标已隐藏**（配合自定义光标）

## Usability 可用性（30%）— 自检清单

### 导航
- [ ] 访客能在 3 秒内理解如何导航（即使导航方式是实验性的）
- [ ] 当前位置有清晰的视觉指示
- [ ] 关键内容在 3 次点击内可达
- [ ] 返回/回到首页有明确路径

### 加载性能
- [ ] 首屏加载 < 3 秒（理想 < 1.5 秒）
- [ ] 有优雅的加载动画（不是白屏等待）
- [ ] 大型资源（3D、视频）延迟加载
- [ ] 动画保持 60fps，无卡顿

### 移动端
**移动端特别重要**——因为大多数用户首先用手机打开网站，Awwwards 评审也优先检查移动端体验。
- [ ] 移动端体验是专门设计的（不是桌面版的缩小版）
- [ ] 3D/WebGL 在移动端有降级方案（2D 替代或简化场景）
- [ ] 触摸操作区域 >= 44px
- [ ] 水平滚动在移动端转为垂直流
- [ ] 光标效果替换为触摸反馈

### Core Web Vitals 目标

| 指标 | 目标值 | 为什么重要 |
|------|--------|-----------|
| LCP（最大内容绘制） | < 1.5s | 用户看到主要内容的时间——超过 2.5s 用户开始失去耐心 |
| CLS（累积布局偏移） | < 0.05 | 页面是否稳定——跳动会让用户感到不可控 |
| INP（交互到绘制） | < 100ms | 点击/输入的响应速度——超过 200ms 会让交互感觉"卡" |
| 页面总大小 | < 3MB | 移动端友好——3MB 在 4G 网络下约需 3-5 秒 |
| Lighthouse Mobile | 90+ 全项 | 综合质量基准 |

## Creativity 创意（20%）— 自检清单

### Wow Moment（签名式时刻）
- [ ] 存在至少一个「让用户停下来截图」的瞬间
- [ ] 这个时刻出现在首屏或首屏之后的 3 秒内
- [ ] 它是「意外 × 情感 × 打磨」的产物，不是纯炫技

### 交互创新
- [ ] 至少有一个交互模式不是从模板复制来的
- [ ] 交互服务于叙事概念（形式统一内容）
- [ ] 微交互覆盖全面（每个可交互元素都有响应）
- [ ] 动画有「灵魂」——能感受到意图，不只是参数

### 技术表达
- [ ] 如果使用 3D/WebGL，它增强了体验而非拖慢了体验
- [ ] 动画节奏经过精心设计（不是所有元素以相同速度运动）
- [ ] 滚动行为有叙事节奏（不是均匀展示）

---

## Content 内容（10%）— 自检清单

### 文案
- [ ] 所有文案都是真实内容（无 Lorem ipsum）
- [ ] 文案有品牌声音和个性（不是通用营销语言）
- [ ] 标题和口号令人印象深刻（不是「欢迎来到我们的网站」）
- [ ] 文案长度适当——每个区块的文案量与视觉比重匹配

### 媒体
- [ ] 图片/视频是原创或专业定制的
- [ ] 媒体质量高（不模糊、不变形、不拉伸）
- [ ] 图片 alt 文本有意义
- [ ] 视频有 poster 帧和懒加载

### 信息架构
- [ ] 内容组织服务叙事，而非功能清单
- [ ] 信息密度适当——不过载也不空洞
- [ ] 每个区块的目的是明确的

## 质量自查流程

### 1. 第一印象测试（5 秒法则）
打开网站全屏截图，只看 5 秒，然后回答：
- 这个网站在讲什么？（叙事清晰度）
- 我想继续看下去吗？（好奇心驱动）
- 它看起来像模板吗？（原创性）

### 2. 10 秒交互测试
在网站上操作 10 秒：
- 导航是否直观？
- 动画是否流畅？
- 有没有让我想截图的瞬间？

### 3. 手机测试
在真实 iPhone/Android 上打开：
- 加载是否快？
- 触摸操作是否舒适？
- 3D 效果是否降级得当？

### 4. Lighthouse 审计
运行 Chrome DevTools Lighthouse（Mobile 模式）：
- Performance ≥ 90
- Accessibility ≥ 80
- Best Practices ≥ 90
- SEO ≥ 90

## 常见质量陷阱（按严重程度排序）

### 致命级（直接毁掉体验）
这些是最容易犯也最致命的错误——如果存在任何一个，其他方面再好也难以弥补：

- **模板感**：一眼看出用的是哪个模板。原因通常是默认组件 + 通用字体 + 标准配色三者的组合。
- **移动端崩坏**：桌面版精美，手机上布局错乱。原因通常是只做了桌面设计，移动端只是简单缩放。
- **5 秒以上加载**：再美的设计用户也看不到。原因通常是未优化的大图片、未压缩的 3D 模型、未懒加载的资源。
- **Lorem ipsum / 占位内容**：直接暴露项目未完成。即使设计精美，没有真实内容也无法评判品质。

### 严重级（设计细节）
- **通用字体（Inter/Roboto）**：字体是第一视觉锚点，通用字体会让整个设计失去个性
- **无意义的 3D**：旋转的几何体与内容无关，这传递的信息是"我在炫技"而非"我有设计意图"
- **导航混乱**：访客不知道如何找到想要的内容，再美也白搭
- **间距不一致**：有些地方紧、有些地方松，没有节奏感——这会暴露设计缺乏系统

### 严重级（CSS 品质信号 — 零成本、不可省略）
以下项目是**零成本**的品质提升——不花开发时间但缺失任何一个都会让网站看起来「没做完」：
- **`::selection` 选中色缺失**：用户选中文字时默认蓝色高亮——这是普通网站最明显的信号之一
- **自定义光标缺失**：默认箭头光标 = 模板网站标配（Awwwards 收藏集 466+ 项）
- **字体抗锯齿缺失**：缺少 antialiased 会让文字看起来"粗糙"
- **自定义滚动条缺失**：默认灰色滚动条与精心设计的页面完全不搭
- **系统光标未隐藏**：自定义光标存在的同时系统默认光标仍然可见
- **表单元素未统一**：::placeholder 和 :focus 使用浏览器默认样式

### 一般级（拉低整体水平）
- **缺少微交互**：按钮悬停没有反馈，链接没有过渡——每个细节都在传递"精致"或"随便"
- **动画卡顿**：掉帧影响体验流畅度，用户会感到不适
- **图片未优化**：模糊或加载缓慢的图片
- **底部设计敷衍**：精美的页面配上一个只有版权信息的 footer——footer 也是品牌触点

### 细节级（精致感的分水岭）
这些是区分"专业"和"业余"的最后 10%：
- **3D 资产细节**：模型法线未烘焙、贴图分辨率不足、阴影缺失
- **过渡微调**：缓动曲线未精心选择，动画感觉"机械"而非"有机"
- **深色模式遗漏**：暗色模式下对比度不足、图片过亮、色彩失真

## 日式设计哲学：Ma（間）

许多顶级工作室（尤其是日本工作室 mount inc.、SHIFTBRAIN）的设计有一个共同的底层哲学——**Ma（間）**。理解这个哲学能帮助你在"少"和"多"之间做出正确的设计决策。

1. **間作为留白**：留白不是「空白」，而是内容的一部分。它让重要元素呼吸，让视觉层次更清晰。

2. **間作为节奏**：元素之间的间距不是随机的，而是像音乐中的休止符——有意识地控制节奏。大间距 = 强调，小间距 = 关联。

3. **間作为克制**：不是每个像素都需要填充内容。少即是多，但「少」中的每一处都必须精雕细琢。选择"少"意味着你对每一个细节都负有更大的责任。

4. **間作为细节执念**：当元素很少时，每个细节都会被放大审视。间距差 2px、字重差 100、缓动曲线差 0.01——这些在密集布局中看不出来，但在克制布局中决定一切。

> **实践指导**：如果你选择「少」的布局，就要对「少」中的每个细节负责。如果你选择「多」的布局，就要确保「多」有清晰的叙事逻辑，而不是元素的堆砌。两种方向都可以达到世界级水准，关键是执行的一致性和深度。

## SOTD 差距诊断

你的网站是 Honorable Mention（6.5）还是 Site of the Day（7.5+）？用这些二元测试诊断：

| 指标 | HM（及格线） | SOTD（头奖线） |
|------|------------|---------------|
| 字体选择 | 趋势字体但配对随意 | 每个字体都有叙事理由 |
| 动画编排 | 所有动画使用同一套 easing | 每个动画的 easing 独立调整 |
| 内容 | 1-2 处使用素材图 | 零库存图片，所有内容原创 |
| 滚动叙事 | 均匀展示各 section | 有节奏变化：快→慢→停→惊喜 |
| 移动端 | "响应式适配"完成 | 独立设计，触摸交互专属体验 |
| 加载体验 | 白屏→内容出现 | 加载过程本身就是品牌叙事 |
| Wow Moment | 有亮点但没让人截图 | 至少 1 个让人截图分享的瞬间 |
| CSS 品质 | 默认选中色/滚动条 | 自定义 ::selection/scrollbar/cursor |
| 概念一致性 | 风格混搭但"看起来不错" | 每个元素都指向同一叙事概念 |
| 打磨深度 | 前 80% 完成 | 最后 20% 的精细打磨完成 |

## 设计陷阱识别

以下特征会让网站看起来像「模板」或「AI 生成的」，而不是「精心设计的」。了解它们的原因比记住规则更重要：

| 特征 | 为什么是问题 |
|------|------------|
| 蓝白配色 + Inter 字体 | 当前最常见的 AI 生成网站特征，用户已经形成了"审美抗体" |
| 紫色到蓝色渐变 hero | 被过度使用到失去了任何视觉冲击力 |
| 三栏卡片 features section | 几乎每个 SaaS 模板都长这样，一秒识别 |
| Bootstrap / Tailwind 默认组件 | 未自定义的框架组件 = 模板感 |
| 通用素材图（Unsplash 等） | 与品牌无关的图片无法建立情感连接 |
| Lorem ipsum / 占位内容 | 直接暴露项目未完成，品质感归零 |
| 无故旋转/缩放的 3D 几何体 | 3D 应该服务于叙事，否则就是"技术演示" |
| 卡片 + 圆角 + 阴影的堆砌 | 需要设计意图来驱动，而非组件的重复 |
| 底部仅一行版权的 footer | 暴露设计系统不完整——footer 也是品牌触点 |

**应该追求的：**
- 超大字体作为视觉锚点（不是图片）——字体本身就是设计
- 有意义的留白（Ma/間 的哲学）——留白是内容而非空白
- 滚动驱动的内容揭示——让用户参与叙事
- 非对称排版和布局——打破模板的均匀感
- 一个令人想截图的 Wow Moment——用户离开后的唯一记忆
