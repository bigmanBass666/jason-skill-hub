# 资源评估框架

本框架定义了如何评估一个免费资源的价值。核心原则：**模型质量 > 免费额度 > 其他一切。**

## 为什么是这个优先级

一个免费的但笨的模型，用户在 agent 工作流中会频繁出错，被迫不断重试，一天消耗 100M token。一个聪明但稍贵的模型，同样的工作可能只需要 10M token。**所以"强模型"本身就是在省钱。** 这不是理论——用户用 Step 3.5 Flash（免费但 agent 弱）和用更好的模型（如果有的话）的实际体验已经证明了这一点。

---

## 评估维度

### 1. 模型质量（Model Quality）— 权重 35%

对于 AI 模型 API，这是最重要的维度。子维度评分均为 1-3 分。

| 子维度 | 1 分 | 2 分 | 3 分 |
|--------|------|------|------|
| Agent/Function Calling 能力 | tool calling 频繁出错，无法完成 agent 工作流 | 偶尔出错，大部分场景可用 | 格式正确不出错，能可靠完成复杂 agent 工作流 |
| 推理质量 | 明显弱于 frontier，经常产生低质量输出 | 中等水平，可完成大部分任务 | 接近 frontier 水平 |
| 上下文长度 | < 32K | 32K-128K | > 128K |

总分范围 3-9。**低于 6 分的模型在 agent 工作流中基本不可用。**

### 2. 时效性机会（Time-sensitivity）— 权重 25%

这条情报本身的价值有多大，取决于它是否有时效性。

| 情况 | 评分 | 说明 |
|------|------|------|
| 匿名/测试模型偷偷上线，可能随时撤下 | 3 | 必须第一时间抓住 |
| 限时优惠活动（明确结束时间） | 2 | 尽早注册占坑 |
| 新平台刚上线，免费层可能缩水 | 1 | 注册占位，但不必急 |
| 已存在较久的稳定免费层 | 0 | 不构成"新情报" |

### 3. 平台价值（Platform Value）— 权重 20%

一个聚合平台的价值远大于单个 API。子维度评分均为 1-3 分。

| 子维度 | 1 分 | 2 分 | 3 分 |
|--------|------|------|------|
| 模型丰富度 | < 10 个模型 | 10-49 个模型 | 50+ 个模型 |
| 切换灵活性 | 需要不同 SDK 或特殊接入 | 部分兼容 | OpenAI 兼容 API，一个 key 切换所有模型 |
| 上新频率 | 很少更新 | 偶尔更新 | 经常上新模型（意味着持续的情报来源价值） |

总分范围 3-9。

### 4. 稳定性（Reliability）— 权重 10%

| 子维度 | 1 分 | 2 分 | 3 分 |
|--------|------|------|------|
| 服务稳定性 | 经常宕机或超时 | 偶尔出问题 | 从不停机，响应稳定 |
| 免费层持久性 | 经常缩水或突然取消 | 偶尔调整 | 从未变更过免费政策 |

总分范围 2-6。

### 5. 可用性（Accessibility）— 权重 10%

| 子维度 | 1 分 | 2 分 | 3 分 |
|--------|------|------|------|
| 国内可访问 | 不可用或需复杂代理 | 需梯子 | 直连可用 |
| 注册门槛 | 需邀请码/绑卡/审核 | 需手机号 | 邮箱即可 |

总分范围 2-6。

---

## 综合评分

综合分 = 质量×0.35 + 时效性×0.25 + 平台价值×0.20 + 稳定性×0.10 + 可用性×0.10

注意：各维度分值范围不同，综合分需要归一化。简化做法：

| 综合评分等级 | 推荐等级 | 行动建议 |
|-------------|---------|---------|
| 高（质量≥7 且 时效性≥2） | 🔥 立即行动 | 马上注册/测试，可能限时 |
| 中高（质量≥6 或 时效性≥2） | 📡 值得关注 | 注册占坑，持续关注 |
| 中（质量≥4） | 📊 可以试试 | 有空时测试 |
| 低 | 跳过 | 不值得花时间 |

## 快速评分模板

对每条新情报，用以下模板快速评分：

```
情报：[一句话描述]
- 模型质量: X/9 (agent=_, 推理=_, 上下文=_)  ← 最重要
- 时效性: X/3 (测试模型=3, 限时=2, 新平台=1, 稳定=0)  ← 第二重要
- 平台价值: X/9 (丰富度=_, 灵活性=_, 上新=_)
- 稳定性: X/6 (服务=_, 持久=_)
- 可用性: X/6 (国内=_, 注册=_)
→ 判定：[🔥/📡/📊/跳过] [行动建议]
```

## 特殊场景评估

### 匿名/测试模型（如 Hunter-alpha）

当发现匿名/未公开宣布的模型时：
- **时效性权重提升到 35%**（可能随时撤下，抢时间是关键）
- **模型质量权重降到 25%**（来不及深入评估，先占了再说）
- **立即标记为 🔥**，即使质量评分不确定
- 评估重点转向：接入方式是否简单？API 兼容性？能否快速跑 agent 工作流测试？
- 输出建议：「立即测试接入，同时在社区跟进该模型的讨论」

### API 中转站（ChatAnywhere 类）

中转站本质上不是资源提供者，而是代理。评估时注意：
- **稳定性风险高**（依赖上游，上游收费中转站也得跟着收费）
- **合规风险**（部分中转站运营方式存疑）
- 但如果确实能用且免费，对用户来说就是有价值的信息
- 标注风险，让用户自行判断
- 重点评估：上游是哪些模型？速率限制？是否支持 OpenAI 兼容格式？

### 模型聚合平台 vs 单个 API

| | 聚合平台（NVIDIA NIM 类） | 单个 API（DeepSeek API 类） |
|---|---|---|
| 模型多样性 | 高（一键切换） | 低（只有自家模型） |
| 发现新模型概率 | 高（平台可能悄悄上新） | 低 |
| 稳定性 | 取决于具体模型 | 取决于该平台 |
| 情报价值 | 高（定期扫平台 = 持续情报源） | 低（看公告即可） |
| 评估重点 | 平台上新频率、模型覆盖 | 单模型质量 |

### 已知资源上的新变化

当已知平台（如 NVIDIA NIM）上出现变化时：
- **新增模型**：即使平台本身已知，新模型也是新情报
- **额度变动**：增加是好消息（📡），缩水是坏消息（💀）
- **模型下线**：坏消息（💀），特别如果用户正在使用
- **稳定性变化**：原来不稳定的模型变稳定了 = 重磅（🔥），反过来 = 预警（⚠️）
