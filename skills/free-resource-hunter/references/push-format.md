# 龙虾推送输出格式

当 skill 被定时触发（如龙虾推送场景）或用户要求简洁输出时，输出必须遵循以下格式。核心原则：**可扫描、可推送、零废话。**

## 标准简报格式

```
📡 免费资源情报雷达 | MM-DD HH:mm
━━━━━━━━━━━━━━━━━━━━

🔥 紧急（N条）

[模型/平台名] 悄悄上线 [平台名]
├ 能力：[参数量/MoE/agent能力]
├ 免费：[额度/限制/时效]
├ 接入：[API格式/链接]
├ 行动：[立即注册/持续关注]

📡 一般（N条）

• [平台] [变动描述] — [影响]
• [平台] [变动描述] — [影响]

📊 已知资源健康

| 平台 | 状态 | 备注 |
|------|------|------|
| NVIDIA NIM | ✅ | Step 3.5 Flash 稳定 |
| OpenRouter | ✅ | 50+ free models |
| ... | ... | ... |

━━━━━━━━━━━━━━━━━━━━
```

## 格式规则

1. **总长度控制在 50 行以内**（推送场景，不是文档场景）
2. **紧急情报最多 3 条**，只放真正值得立即行动的
3. **一般动态最多 10 条**，每条一行
4. **健康表只列用户正在使用或关注的主要平台**（5-8 个）
5. **不用 emoji 做标题装饰**，但 🔥📡📊📭 作为分类标记保留
6. **每条情报必须有可操作的信息**（链接、注册方式、接入方式）
7. **标注时效性**：如果某个优惠"可能限时"，明确写出来
8. **绝不含废话**：不需要开头寒暄、结尾总结、解释说明

## 评分标签

| 标签 | 含义 | 何时使用 |
|------|------|---------|
| 🔥 | 紧急/重磅 | 新上线的强模型、限时免费、重大政策变更、匿名/测试模型 |
| 📡 | 一般动态 | 额度微调、新模型上线但能力一般、平台更新 |
| 📊 | 健康快照 | 已知资源的当前状态（每次简报都带） |
| ⚠️ | 预警 | 免费层可能缩水、平台不稳定、需要关注 |
| 💀 | 坏消息 | 免费层取消、平台关闭、模型下线 |
| 📭 | 无变化 | 本次扫描未发现任何新情报 |

## 情报时效性标记

在每条情报末尾，用以下标记告知用户这条情报的"保鲜期"：

| 标记 | 含义 |
|------|------|
| `[永久]` | 长期可用的免费资源 |
| `[限时]` | 有明确的结束时间 |
| `[测试期]` | 正在测试，可能随时撤下 |
| `[不稳定]` | 可用性波动大 |
| `[需验证]` | 情报来源不够可靠，建议用户自行确认 |

## 日常对话模式

当用户不是定时推送而是直接对话时，输出格式可以更灵活：
- 不需要严格遵循上述 50 行限制
- 但仍然使用 🔥📡📊⚠️💀 分类标记
- 仍然标注时效性
- 可以展开详细说明（推送模式不能展开，对话模式可以）
- 可以包含技术分析、对比、个人建议等内容
