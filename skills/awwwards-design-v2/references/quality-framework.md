# 设计质量框架

基于 Awwwards 2024–2026 年度获奖网站逆向工程，提炼出的设计质量自检体系。这四个维度决定了你的网站是否具备「世界级设计」的水准——用它们作为自查清单，而非参赛标准。

---

## 质量优先级矩阵

当一个设计决策需要取舍时，用这个优先级指导投入方向。权重来自对数百个获奖网站的分析——视觉和可用性是绝对大头，创意是差异化因素，内容是信任基础。

| 维度 | 权重 | 核心关注 | 致命缺陷 |
|------|------|----------|----------|
| **Design 设计** | 40% | 视觉层次、字体品位、色彩克制、间距节奏、原创性 | 模板感、通用字体（Inter/Roboto）、素材库图片 |
| **Usability 可用性** | 30% | 导航直观、加载 <3s、移动端体验、Core Web Vitals | 移动端崩坏、5s+ 加载、导航混乱 |
| **Creativity 创意** | 20% | 一个签名式「Wow Moment」、交互创新、形式统一内容 | 抄袭模式、创意损害可用性 |
| **Content 内容** | 10% | 真实文案、品牌声音、原创媒体 | Lorem ipsum / 占位内容、机器翻译文案、素材图 |

> **为什么是这个权重？** 因为一个好看但难用的网站不会给人留下好印象（设计 40% + 可用性 30% = 70%）。一个好看又好用但毫无新意的网站只是「还不错」而非「令人难忘」（需要创意 20% 来突破）。而内容（10%）是最容易被忽视但能显著提升整体品质感的环节。

## Design 设计（40%）— 自检清单

### 视觉层次
- [ ] 页面打开的瞬间，眼睛知道先看哪里（明确的视觉锚点）
- [ ] 信息层级不超过 3 级（主 → 次 → 辅助），每一级在字号/颜色/粗细上有明显区分
- [ ] 没有任何两个同等重要的元素在争夺注意力

### 字体
- [ ] 展示字体独特且有意义（不是 Inter/Roboto/Arial/Poppins）
- [ ] Hero 字号极端化（`clamp(4rem, 12vw, 14rem)`），字体本身就是视觉元素
- [ ] 字体配对有对比且和谐（展示字体 vs 正文字体风格差异明显）
- [ ] 行高、字间距经过调整（Hero 行高 0.85-0.92，字间距 -0.03em 到 -0.05em）

### 色彩
- [ ] 调色板不超过 3 色（背景 + 文字 + 1个强调色）
- [ ] 强调色使用克制，仅出现在关键交互点和品牌元素上
- [ ] 色彩选择服务于品牌叙事，而非随意搭配
- [ ] 暗色模式下 3D 元素和渐变效果得到充分展示

### 间距与节奏
- [ ] 每个间距都是刻意设计的（不是默认 margin/padding）
- [ ] 章节间距使用 clamp 实现流式缩放
- [ ] 留白是有意义的（Ma/間 的哲学）——留白是内容而非空白
- [ ] 元素对齐使用一致的网格系统

### 原创性
- [ ] 至少有一个视觉元素不是来自模板或素材库
- [ ] 图形元素服务于叙事概念
- [ ] 整体视觉感觉「做过设计」而非「选过模板」
- [ ] 与同类型网站放在一起，能一眼区分出来

## Usability 可用性（30%）— 自检清单

### 导航
- [ ] 访客能在 3 秒内理解如何导航（即使导航方式是实验性的）
- [ ] 当前位置有清晰的视觉指示
- [ ] 关键内容在 3 次点击内可达
- [ ] 返回/回到首页有明确路径

### 加载性能
- [ ] 首屏加载 < 3 秒（理想 < 1.5 秒）
- [ ] 有优雅的加载动画（不是白屏等待）
- [ ] 大型资源（3D、视频）延迟加载
- [ ] 动画保持 60fps，无卡顿

### 移动端
- [ ] 移动端体验是专门设计的（不是桌面版的缩小版）
- [ ] 3D/WebGL 在移动端有降级方案（2D 替代或简化场景）
- [ ] 触摸操作区域 >= 44px
- [ ] 水平滚动在移动端转为垂直流
- [ ] 光标效果替换为触摸反馈

### Core Web Vitals 目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| LCP（最大内容绘制） | < 1.5s | 用户看到主要内容的时间 |
| CLS（累积布局偏移） | < 0.05 | 页面是否稳定、不跳动 |
| INP（交互到绘制） | < 100ms | 点击/输入的响应速度 |
| 页面总大小 | < 3MB | 移动端友好 |
| Lighthouse Mobile | 90+ 全项 | 综合质量 |

## Creativity 创意（20%）— 自检清单

### Wow Moment（签名式时刻）
- [ ] 存在至少一个「让用户停下来截图」的瞬间
- [ ] 这个时刻出现在首屏或首屏之后的 3 秒内
- [ ] 它是「意外 × 情感 × 打磨」的产物，不是纯炫技

### 交互创新
- [ ] 至少有一个交互模式不是从模板复制来的
- [ ] 交互服务于叙事概念（形式统一内容）
- [ ] 微交互覆盖全面（每个可交互元素都有响应）
- [ ] 动画有「灵魂」——能感受到意图，不只是参数

### 技术表达
- [ ] 如果使用 3D/WebGL，它增强了体验而非拖慢了体验
- [ ] 动画节奏经过精心设计（不是所有元素以相同速度运动）
- [ ] 滚动行为有叙事节奏（不是均匀展示）

---

## Content 内容（10%）— 自检清单

### 文案
- [ ] 所有文案都是真实内容（无 Lorem ipsum）
- [ ] 文案有品牌声音和个性（不是通用营销语言）
- [ ] 标题和口号令人印象深刻（不是「欢迎来到我们的网站」）
- [ ] 文案长度适当——每个区块的文案量与视觉比重匹配

### 媒体
- [ ] 图片/视频是原创或专业定制的
- [ ] 媒体质量高（不模糊、不变形、不拉伸）
- [ ] 图片 alt 文本有意义
- [ ] 视频有 poster 帧和懒加载

### 信息架构
- [ ] 内容组织服务叙事，而非功能清单
- [ ] 信息密度适当——不过载也不空洞
- [ ] 每个区块的目的是明确的

## 质量自查流程

### 1. 第一印象测试（5 秒法则）
打开网站全屏截图，只看 5 秒，然后回答：
- 这个网站在讲什么？（叙事清晰度）
- 我想继续看下去吗？（好奇心驱动）
- 它看起来像模板吗？（原创性）

### 2. 10 秒交互测试
在网站上操作 10 秒：
- 导航是否直观？
- 动画是否流畅？
- 有没有让我想截图的瞬间？

### 3. 手机测试
在真实 iPhone/Android 上打开：
- 加载是否快？
- 触摸操作是否舒适？
- 3D 效果是否降级得当？

### 4. Lighthouse 审计
运行 Chrome DevTools Lighthouse（Mobile 模式）：
- Performance ≥ 90
- Accessibility ≥ 80
- Best Practices ≥ 90
- SEO ≥ 90

## 常见质量陷阱（按严重程度排序）

### 致命级（直接毁掉体验）
- 模板感：一眼看出用的是哪个 WordPress/Webflow 模板
- 移动端崩坏：桌面版精美，手机上布局错乱
- 5 秒以上加载：再美也没有用户愿意等
- Lorem ipsum：说明项目还没准备好上线

### 严重级（大幅降低品质感）
- 通用字体（Inter/Roboto）：没有视觉个性
- 无意义的 3D：旋转的几何体与内容无关
- 导航混乱：访客不知道如何找到想要的内容
- 间距不一致：有些地方紧、有些地方松，没有节奏感

### 一般级（拉低整体水平）
- 缺少微交互：按钮悬停没有反馈，链接没有过渡
- 动画卡顿：掉帧影响体验流畅度
- 图片未优化：模糊或加载缓慢的图片
- 底部设计敷衍：精美的页面配上一个只有版权信息的 footer

### 细节级（精致感的分水岭）
- 光标效果缺失：自定义光标是最小投入最大回报的设计元素
- 加载动画缺失：白屏等待让人焦虑
- 字体渲染：缺少 `-webkit-font-smoothing: antialiased`
- 滚动体验：原生滚动 vs Lenis 平滑滚动，差距明显

## 日式设计哲学：Ma（間）

许多顶级获奖工作室（尤其是日本工作室 mount inc.、SHIFTBRAIN）的设计有一个共同的底层哲学——**Ma（間）**：

1. **間作为留白**：留白不是「空白」，而是内容的一部分。它让重要元素呼吸。
2. **間作为节奏**：元素之间的间距不是随机的，而是像音乐中的休止符——有意识地控制节奏。
3. **間作为克制**：不是每个像素都需要填充内容。少即是多，但「少」中的每一处都必须精雕细琢。
4. **間作为细节执念**：当元素很少时，每个细节都会被放大审视。间距差 2px、字重差 100、缓动曲线差 0.01——这些在密集布局中看不出来，但在克制布局中决定一切。

> 将这个哲学应用到你的设计中：如果你选择「少」，就要对「少」中的每个细节负责。如果你选择「多」，就要确保「多」有叙事逻辑。
