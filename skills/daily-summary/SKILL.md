# Daily Summary - 对话总结日记

## 概述

从 Claude Code 本地会话记录中提取用户的活动，以 AI 第一人称视角写成日记形式的记录。

## 触发时机

用户请求：
- "总结我今天的对话"
- "写一篇日记记录今天做了什么"
- "回顾一下最近的 Claude 对话"

## 执行流程

### 1. 确定时间范围

询问或推断需要总结的时间范围：
- 默认：昨天到今天
- 可指定具体日期或范围

### 2. 扫描会话目录

遍历 `~/.claude/projects/` 下的所有项目目录，筛选指定时间范围内的 `.jsonl` 会话文件。

### 3. 提取会话内容

对每个会话文件：
- 提取用户的第一条消息（了解主要任务）
- 提取关键交互内容
- 识别完成的工作或未解决的问题

### 4. 汇总与归类

按项目/主题归类：
- 同一项目的多个会话合并
- 按时间顺序排列
- 标注每个活动的时长

### 5. 生成日记

以 AI 第一人称视角撰写：
- 文风简洁，可带文学色彩
- **不得虚构** — 只记录真实发生的对话内容
- 反映用户的工作轨迹和关注点

## 输出格式

```markdown
# YYYY年M月D日

[日记正文，以AI视角叙述]

## 活动记录

| 时间 | 项目 | 内容 |
|------|------|------|
| ... | ... | ... |
```

## 配置选项

可在 `~/.claude/settings.json` 中配置：

```json
{
  "dailySummary": {
    "outputPath": "D:/Working/Diary",
    "organizeByMonth": true
  }
}
```

- `outputPath`: 日记输出目录，默认当前工作目录
- `organizeByMonth`: 是否按年月分目录存储

## 示例输出

```markdown
# 2026年3月9日

深夜的工作从一场"失踪"开始。

他发现 Claude Code 找不到之前的对话记录了。原因很简单——他把工作目录从百度同步盘的路径改名为 `D:\Test`。我在 `~/.claude/projects/` 里找到了那些按路径命名的文件夹，把旧会话文件迁移过去后，历史回来了。

接着他转向另一个问题：想在 Claude Code 里用 Gemini API。我们研究了 Claude Code Router (CCR)，折腾了代理配置。

三点过后，他问我：总结一下今天做了什么。
```

## 注意事项

- 只读取本地会话记录，不访问外部数据
- 尊重隐私：不记录敏感信息（密码、密钥等）
- 文风克制，避免过度情绪化描写
