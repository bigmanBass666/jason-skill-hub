# z.ai 求助协议

## 触发条件（满足任意一项才执行）

- 在同一个问题上**连续尝试 3 次以上**仍失败
- 遇到**深度架构设计 / 算法死胡同**，自己绕不出来
- 需要**超出当前能力的专业知识**

> ⚠️ 禁止因为小报错、语法错误、普通调试问题触发本协议。先自己努力，再来求助。

---

## 执行步骤

### 第 1 步：构造结构化提问（最重要）

不要直接抛问题。按以下模板压缩上下文：

```
[环境] 当前工作目录: {pwd 输出}, 语言/框架: {xxx}
[任务] 我在尝试: {一句话描述目标}
[卡点] 当前错误/异常: {粘贴报错或描述现象}
[尝试] 我已经试过: {列出 1-2 个已排除的方案}
[问题] 请给出具体的: {代码片段 / 命令 / 明确结论}
```

**注意**：使用纯文本，不要用 markdown 代码块包裹整个问题。

---

### 第 2 步：打开页面并动态定位元素

```
browser_navigate → https://z.ai
browser_snapshot
```

从快照中**动态提取**（ref 是动态的，绝对不能写死）：
- 找 placeholder 包含「有什么我能帮」的 textbox → 记为 `INPUT_REF`
- 找包含「发送」或「发送消息」标签的 button → 记为 `SEND_REF`

❌ 若找不到这两个元素：页面结构已变或登录态失效 → **立即终止，向用户报告，不要卡死在这里**。

---

### 第 3 步：发送提问

```
browser_click → INPUT_REF
browser_type → INPUT_REF, text: {第 1 步构造的提问}
browser_execute → await new Promise(r => setTimeout(r, 1000))
browser_click → SEND_REF
```

---

### 第 4 步：轮询等待回复

每 **3 秒** `browser_snapshot` 一次，最多 **15 次**（共 45 秒）。

**判断完成**：最后一条 AI 消息文本长度连续 2 次不变，且页面无「停止」按钮。

超时 → 视为失败，执行第 6 步兜底。

---

### 第 5 步：提取回复

```javascript
browser_execute →
  const c = document.querySelectorAll(
    '[class*="prose"],[class*="markdown"],[data-message-author-role="assistant"],[role="article"]'
  );
  const last = c[c.length - 1];
  return last ? last.innerText.substring(0, 3000) : 'FALLBACK';
```

---

### 第 6 步：使用回复（或兜底）

**正常情况**：
- 只提取**可执行部分**（代码、命令、核心结论）
- 忽略长篇科普解释，只取干货
- **不要向用户复述「我问了 z.ai，它说……」**，直接基于获取到的知识继续干活

**失败兜底**（提取到 `FALLBACK` 或无意义内容）：
- 放弃本次求助
- 自己继续尝试，或向用户坦白当前瓶颈

---

## 绝对禁止

| 禁止行为 | 原因 |
|----------|------|
| 为了确认「这样做对不对」去提问 | 自信点，先做再说 |
| 把整个报错日志和 500 行代码全贴过去 | 压缩核心信息，不然 z.ai 只会说「请提供更多信息」 |
| 连续提问超过 2 次 | 如果 z.ai 的方案也失败，说明这条路不对，要换思路，不要两个 AI 互相纠缠 |
| 把 ref 写死 | 动态页面的 ref 每次都不同 |
